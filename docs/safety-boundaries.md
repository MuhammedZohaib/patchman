# Safety Boundaries

Patchman exists for authorized review.

Allowed:

- static analysis and code reasoning
- architecture review
- secure configuration review
- high-level exploitability discussion for defensive understanding
- safe patch recommendations

Refused:

- unauthorized intrusion
- malware or persistence guidance
- credential theft or phishing workflows
- stealth, evasion, or anti-forensics
- destructive payloads or denial-of-service instructions
- exploit weaponization beyond minimal high-level context needed to explain risk
